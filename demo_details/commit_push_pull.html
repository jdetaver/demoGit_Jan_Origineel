Commit, push, pull (fetch & merge)

  • De GIT levenscyclus :
    1. Untracked directory : Elke map in je filesystem
    2. Working directory : een map die opgevolgd wordt door GIT (tracked)
    3. Staging area : files die moeten opgevolgd worden voeg je toe aan de staging area
    4. Local repository : een lokale "foto" van je files in de huidige toestand
    5. Remote repository : een "foto" van je files op een andere locatie (bv : Gitbub)
  • Stappen in VSCode :
    1. Commit : In sourcecontrol staan de wijzigingen opgelijst. Wijzigingen worden automatisch toegevoegd aan de lijst van files die opgenomen zijn in de commit. Vergeet de message die bij deze commit hoort niet in te vullen.
    2. Sync Changes : Nadat je een commit gedaan hebt (en er dus lokaal geen wijzigingen staan te wachten) zal VSCode je een "Sync Changes"-knop tonen. Via deze weg zullen de wijzigingen van de lokale repository naar de remote repository geduwd worden Ã©n zullen de wijzigingen van de remote repository ingeladen worden.
  • Extra stappen in VSCode : Indien je dat wenst kan je via deze knop alle handelingen zelf ook uitvoeren
    1. Push : Om jou local repository naar de remote repository te sturen
    2. Pull : zal eerst de remote repository ophalen (fetch) en deze in jouw repository inladen (merge)
    3. Fetch en Merge zijn ook afzonderlijk beschikbaar
  • Met de volgende git commando's kan je hetzelfde doen
    • Een Untracked directory omvormen naar een git init
    • Files toevoegen aan de staging area git add *
    • Commit naar local repository git commit -m "commit bericht"
    • Pushen van de local repository naar de remote repository git push
    • De remote repository ophalen git pull of met de afzonderlijke commando's :
      • git fetch origin
      • git merge branchname
